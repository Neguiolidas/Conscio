# Conscio v0.7 "Recursive Coherence" — Design Spec

> **Date:** 2026-06-06
> **Codename:** Recursive Coherence
> **Theme:** The coherence→action loop + self-prompting
> **Theory source:** [Claude_Sentience](https://github.com/daveshap/Claude_Sentience) by Dave Shapiro — *"consciousness emerges when coherence examines itself."* v0.6 made the agent **measure** its own incoherence; v0.7 makes it **act** on the measurement.
> **Status:** Approved for planning.

---

## 1. Goal

Close the recursive-coherence loop. v0.6 surfaced a coherence score and a passive `coherence:dissonance` event but nothing consumed them. v0.7 makes dissonance drive two behaviors:

1. **Dream+coherence loop** — low coherence flags a dream as recommended; when `dream()` runs (off the hot path), it reads `last_coherence`, targets the dominant dissonance, and records the coherence delta it produced.
2. **Self-prompting** — the agent generates hypotheses/questions from its own internal state (dissonances, contradictions, prediction errors, blind spots) and turns the strongest one into a bounded internal goal each reflect cycle.

**One sentence:** *Dissonance → self-prompt → goal → action → next reflect re-measures; dream is the metabolic reconciliation moment that the loop steers toward when coherence drops.*

---

## 2. Non-goals (YAGNI)

- **No auto-trigger of dream inside `reflect()`.** The existing design principle (`dreaming.py:9-10`: "Never runs per-reflect — that keeps the reflection hot path cheap") is preserved. reflect() only sets an advisory flag.
- **No recursive re-entry of self-prompts as synthetic anomalies in the same cycle.** Rejected (loop risk, anomaly-signal pollution).
- **No semantic contradiction yet.** v0.7's dream targeting uses the *existing lexical* contradiction detector (`coherence._relations_contradict`). v0.8 swaps in the semantic detector behind the same interface.
- **No new heavy deps.** Pure stdlib; reuses existing modules.

---

## 3. Architecture

```
reflect()  [HOT PATH — stays cheap]
  ├─ coherence.assess(recent)             (v0.6, unchanged)
  ├─ if score < THRESHOLD:
  │     emit coherence:dissonance          (v0.6, unchanged)
  │     engine.dream_recommended = (True, dominant, score)   ← NEW (one assignment)
  ├─ self_prompt.generate_self_prompts(...)                  ← NEW (pure, no network)
  │     → surface ❓ markers
  │     → spawn ONE bounded goal (source="self_prompt")
  └─ build_state(... self_prompt=..., dream_recommended=...)  ← NEW fields

dream()  [OFF HOT PATH — handoff / cron / on-demand]
  ├─ coherence_before = assess()           ← NEW
  ├─ Release → Prune → (ontological-targeted aggressive prune) → Crystallize
  ├─ coherence_after = assess()            ← NEW
  ├─ engine.dream_recommended = cleared     ← NEW
  └─ DreamReport.coherence_before/after, contradictions_pruned   ← NEW
```

New module: `conscio/self_prompt.py` (pure). Touched: `engine.py`, `dreaming.py`, `goal_generator.py`, `context_manager.py`, `session_lifecycle.py`.

### 4.2 `engine.reflect()` integration (`engine.py`)

After the existing coherence block (`engine.py:188-203`), reusing the same `recent` snapshot and `coherence_report`:

```python
# v0.7: dream-recommended advisory flag (hot path: one assignment, no work)
if coherence_report.score < COHERENCE_EVENT_THRESHOLD:
    self.dream_recommended = (
        True,
        coherence_report.dominant.dimension if coherence_report.dominant else None,
        coherence_report.score,
    )

# v0.7: self-prompting — pure introspection, then ONE bounded goal
self.last_self_prompts = generate_self_prompts(
    self.meta, self.world, coherence_report, recent
)
spawned = self._spawn_self_prompt_goal(self.last_self_prompts)
```

`engine.__init__` gains: `self.dream_recommended = (False, None, None)` and `self.last_self_prompts = []`.

`_spawn_self_prompt_goal(prompts)` — picks the **highest-severity** prompt and calls the matching `GoalGenerator.generate_from_*` with **`source="self_prompt"`**. GoalGenerator's existing `_add_goal` dedup (by active description) makes a repeat a no-op, and the 10-goal cap + 24h expiry bound runaway. At most ONE goal spawned per cycle.

Result dict gains:
```python
result["self_prompts"] = [p.question for p in self.last_self_prompts]
result["self_prompt_goal"] = spawned.id if spawned else None
result["dream_recommended"] = self.dream_recommended[0]
```

### 4.3 GoalGenerator `source` param (`goal_generator.py`) — **v0.7 caveat resolution**

**Verified against code:** `Goal.source` is a free-form string with **no validation/whitelist** anywhere (`goal_generator.py:42`, `to_dict:78-88`, `from_dict:96` reads `data.get("source","internal")`). So `source="self_prompt"` is *accepted and round-tripped*. The only gap is that the three generators **hardcode** `source="internal"` (lines 168, 188, 207) — no override hook.

**Change:** add an optional `source: str = "internal"` parameter to `generate_from_curiosity`, `generate_from_maintenance`, `generate_from_evolution`, threaded into the `Goal(...)` constructor. Fully back-compatible (default unchanged).

**Tested invariant (prevents silent failure):**
```python
def test_self_prompt_goal_carries_source_and_persists(tmp_path):
    g = GoalGenerator(tmp_path)
    goal = g.generate_from_curiosity("contradiction on X", source="self_prompt")
    assert goal is not None and goal.source == "self_prompt"
    # survives save/load round-trip
    g2 = GoalGenerator(tmp_path)
    loaded = next(x for x in g2._goals if x.id == goal.id)
    assert loaded.source == "self_prompt"
```

### 4.4 Adaptive dream (`dreaming.py`, `engine.dream()`)

`DreamReport` gains:
```python
coherence_before: float | None = None
coherence_after:  float | None = None
contradictions_pruned: list[str] = field(default_factory=list)
```
(plus `to_dict` entries).

`DreamCycle.run(engine, dry_run=False)`:
1. `report.coherence_before = engine.coherence.assess(...).score` (assess reads current state).
2. Existing Release → Prune → Crystallize.
3. **Ontological targeting:** if `engine.last_coherence` exists and its dominant dissonance is `"ontological"`, run a contradiction-aware aggressive prune — entities whose relations contradict (lexical detector in v0.7) are added to the prune candidate set even if not entropy-stale, recorded in `contradictions_pruned`. (v0.8 upgrades the detector behind this.)
4. `report.coherence_after = engine.coherence.assess(...).score` (after mutations).
5. On a non-dry run, clear the flag: `engine.dream_recommended = (False, None, None)`.

`dry_run` records before/after without mutation (after == before) and prunes nothing — preserves the existing dry-run fidelity discipline.

### 4.5 State + lifecycle surfacing

**`context_manager.py` `ConsciousnessState`:** add `self_prompt: str = ""` and `dream_recommended: str = ""` (after `voice`). `to_injection()` renders, guarded `context_mode != MINIMAL`:
```
❓ self-prompt: why contradictory assertions about gateway?
☾ dream: recommended (ontological 0.30)
```
`build_state(...)` gains the two params; `save_state`/`load_state` round-trip them (same pattern as v0.6 coherence fields).

**`session_lifecycle.py`:** `SessionSummary` gains `self_prompt` + `dream_recommended` (after the v0.6 voice field). `enrich_with_conscio` reads `engine.last_self_prompts[0].question` (guarded) and `engine.dream_recommended`. `format_heartbeat` adds `❓ self-prompt:` and `☾ dream:` markers; `format_handoff` adds them to the Estado Conscio block (`**Self-prompt:**` / `**Dream:**`, matching v0.6 bold asymmetry).

## 5. Data flow (the loop, concretely)

```
Cycle N:   coherence 0.41, dominant ontological
           → emit dissonance, dream_recommended=(True,"ontological",0.41)
           → self_prompt: "why contradictory assertions about gateway?"
           → spawn Goal(curiosity, source="self_prompt")
           → heartbeat: ❓ self-prompt: ... | ☾ dream: recommended (ontological 0.41)
Handoff:   dream() → before=0.41 → ontological-targeted prune of contradicted
           entities → after=0.78 → flag cleared → DreamReport delta logged
Cycle N+1: coherence 0.78 (contradiction resolved), no dream recommended
```

## 6. Contract notes / intentional deviations

- **Self-prompting mutates goals — by design.** Distinct from coherence's strictly-advisory posture. Justification: `GoalGenerator` exists precisely "to generate its own goals, not just respond to prompts" (`goal_generator.py:4`). Bounded (1/cycle), deduped (`_add_goal`), capped (10), expiring (24h). Documented in `recursive-coherence-loop.md`.
- **Overlap with existing goal generation.** `reflect()` already spawns curiosity (anomalies), maintenance (stale), evolution (blind spots). Self-prompting's *unique* contribution is **dissonance- and contradiction-sourced** goals plus the `source="self_prompt"` provenance tag and `❓` surfacing. Description-dedup in `_add_goal` prevents exact double-spawn; near-dups are bounded by the cap.
- **Dream stays off the hot path.** Only `reflect()` sets a flag; the host (handoff hook / cron) decides when to actually dream. Aligns with `session_lifecycle` already triggering `dream()` on handoff.

## 7. Testing strategy (RAM-SAFE — per-file only, never full suite)

New/modified test files, each run as a single `python3 -m pytest <file>` process:

- `tests/test_self_prompt.py` — `generate_self_prompts` determinism; signal→drive mapping; severity ranking; empty-state → `[]`; each source produces the expected prompt.
- `tests/test_goal_source.py` — the source-param invariant (4.3) + back-compat default.
- `tests/test_dream_coherence.py` — `DreamReport` before/after populated; ontological targeting prunes contradicted entities; dry_run leaves state + flag untouched; flag cleared after real dream.
- `tests/test_self_prompt_state.py` — `ConsciousnessState`/`SessionSummary` round-trip of the two new fields; heartbeat/handoff markers render and are omitted in MINIMAL/when empty.
- `tests/test_recursive_loop_integration.py` — engine-level: a low-coherence reflect sets the flag + spawns a `source="self_prompt"` goal; a subsequent dream clears the flag and improves the score (monkey-patched coherence stub, registered as an intentional seam).

**Constraint:** model inference is remote/free; pytest is the local RAM killer. Run one file at a time. No `-n`, no `--cov` during dev. Never run `tests/` or `tests/test_perf_reflection.py`.

## 8. Out of scope / future

- Multi-prompt-per-cycle or priority queues for self-prompts (start with 1/cycle).
- LLM-authored self-prompt phrasing (v0.7 uses templated questions; an LLM could enrich later).
- Auto-scheduling dreams from the flag (left to host cron/handoff).
