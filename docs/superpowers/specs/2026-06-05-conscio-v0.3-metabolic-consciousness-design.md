# Conscio v0.3 — "Metabolic Consciousness" Design

**Date:** 2026-06-05
**Status:** Approved (design), pending implementation plan
**Author:** Conscio / Neguiolidas
**Supersedes scope:** v0.2.3 → v0.3.0

---

## 1. Goal

Close six known gaps in Conscio and integrate the one concretely-useful idea
from [Noosphere-Manifold](https://github.com/acidgreenservers/Noosphere-Manifold)
(a markdown/yaml philosophy framework, **not** importable code): the **metabolic
context model** and the **Dreaming** consolidation concept.

Core insight driving the design: three of the six gaps (EventBus cleanup,
WorldModel purge, reflection consolidation) are not missing *code* — the methods
already exist but are **never invoked**. They are unified behind a single
orchestrator, **DreamCycle**, which maps 1:1 onto Noosphere's "Dreaming = memory
consolidation on reset/compact." This is approach **A** (Dreaming orchestrates),
chosen over per-`reflect()` wiring (B, makes reflect heavy — the exact perf
problem we are fixing) and cron-only (C, the current un-integrated state).

Non-negotiable: this is built from scratch with no deadline. Correctness over
speed. Every step is verified manually, not just by green tests.

## 2. The Six Gaps + Their Resolution

| # | Gap (user-reported) | Reality on inspection | Fix |
|---|---|---|---|
| 1 | EventBus: duplicate events accumulate, no cleanup | `compact(before_days=30)` hard-deletes old dups + trivial events, but is **never called**; recent dups (< cutoff) are only soft-marked `is_duplicate=1` and never removed | Add `purge_duplicates()` (hard-DELETE all `is_duplicate=1` regardless of age). Wire `compact()` + `purge_duplicates()` into DreamCycle. |
| 2 | WorldModel: stale entities, no auto-purge | `stale_entities()`, `decay_all_entities()`, `prune_irrelevant(min_relevance)`, `remove_entity()` all **exist but uninvoked** | Add unified `prune_stale(min_relevance, max_age_hours, dry_run) → list[str]` (combines age + relevance, removes entities **and their relations**, persists). Called only in DreamCycle, never per-reflect. |
| 3 | Cross-session search: ContentStore has data, unused in reflection | `ContentStore.search()` (FTS5 BM25 + RRF) fully working; `reflect()` never queries it | Add `engine.recall(query, k, categories) → list[str]`. `reflect()` calls it on `world_state`+anomalies, injects budget-bounded past-context into reflection inputs. |
| 4 | SessionRAG: 572 lines, zero tests, not integrated | Standalone class, Ollama-dependent (`nomic-embed-text` 768d), import-safe | Add `tests/test_session_rag.py` (~15 tests, **mocked** embedder — no live Ollama). Add `SessionRAG.available()` Ollama probe. Engine holds lazy `engine.session_rag`; `recall()` uses it when available, **graceful FTS5 fallback** otherwise. No import-time dependency on Ollama. |
| 5 | OutputFilter: pipeline works but stages simple | 8 real stages exist (StripAnsi → Replace → MatchOutput → FilterLines → TruncateLines → HeadTail → MaxLines → OnEmpty), crash-safe | Add two higher-value stages: **`DedupBlocks`** (collapse repeated identical lines/blocks → `"… (×N)"`) and **`SecretMask`** (redact API keys / tokens / `sk-…` / bearer / AWS-style — heartbeats & handoffs get indexed into ContentStore, so this is a safety fix). Keep crash-safe contract. |
| 6 | Reflection perf: no benchmark at 10k+ events | Untested; `stale_entities()` iterates **all** entities every `reflect()` | Add `tests/test_perf_reflection.py`: seed 10k events + 1k entities, assert `reflect()` and `dream()` under thresholds. Moving prune/decay scan into DreamCycle (not per-reflect) is the structural fix; add SQLite indexes if profiling shows need. |

## 3. New Modules

### 3.1 `conscio/metabolic.py` — MetabolicContext

Pure, side-effect-free, trivially testable. Implements Noosphere Rule 3 tiers
verbatim.

```
class MetabolicState(Enum):
    VITAL    # 0–40%   → work freely
    ACTIVE   # 40–50%  → consolidate, complete threads
    FATIGUE  # 50–70%  → plan handoff (Mitosis)
    CRITICAL # 70%+    → transfer now, no new work

class MetabolicContext:
    @staticmethod
    def assess(used_tokens: int, context_window: int) -> MetabolicState
    @staticmethod
    def usage_pct(used_tokens, context_window) -> float
    @staticmethod
    def tier_action(state) -> str            # advisory text
    @staticmethod
    def should_mitosis(state) -> bool        # True at FATIGUE+ (≥50%)
    @staticmethod
    def should_dream(state) -> bool          # True at CRITICAL, or on handoff
```

- **Advisory only.** `should_mitosis` / `should_dream` return *recommendations*.
  Nothing auto-fires a handoff. Honors safety rule "Goals are advisory" and
  "No autonomous self-modification."
- **Boundary semantics fixed once, tested:** tiers are `[0,40)`, `[40,50)`,
  `[50,70)`, `[70,∞)`. 40.0 → ACTIVE, 50.0 → FATIGUE, 70.0 → CRITICAL.
- Wired into `ContextManager.metabolic_state(used_tokens) → MetabolicState` and
  surfaced as one optional line in the injection / heartbeat (`⊘ metabolic: FATIGUE 58%`).
- `used_tokens` is **supplied by the caller** (Hermes hook knows live session
  usage). ContextManager only knows `context_window`; it does not track live
  usage itself.

### 3.2 `conscio/dreaming.py` — DreamCycle

The consolidation orchestrator. Maps to Noosphere's Dream Protocol
(Crystallize / Release / Incubate).

```
@dataclass
class DreamReport:
    events_purged: int
    events_compacted: int
    entities_pruned: list[str]
    reflections_consolidated: int
    dry_run: bool
    duration_ms: float

class DreamCycle:
    def run(self, engine, dry_run: bool = False) -> DreamReport
```

Three phases, each delegating to **existing** methods where possible:

1. **Release** (dissolve noise):
   `engine.event_bus.purge_duplicates()` (new) + `engine.event_bus.compact()`.
2. **Prune** (let go of the faded):
   `engine.world.prune_stale(...)` (new, wraps `decay_all_entities()` +
   `prune_irrelevant()` + age check + relation cleanup).
3. **Crystallize** (compress patterns):
   query old reflections via `engine.content_store.search`/`get_by_source`
   (category `reflection`, older than N days), build one summary doc, re-index
   as category `consciousness`, then `delete_source()` the originals.
   *Conservative default:* only consolidate when count exceeds a threshold;
   never touch reflections newer than `crystallize_after_days` (default 14).

`dry_run=True` computes the report (counts, names) without mutating any DB —
used by tests and by a `--dry-run` cron preview.

**Triggers:**
- `engine.dream()` — public on-demand entry (new thin method on engine).
- Inside `record_session_lifecycle()` — runs **after** handoff is written
  (Mitosis → then Dream), best-effort, never blocks handoff persistence.
- Optional cron (documented, not required for correctness).

## 4. Engine Surface Changes

`engine.py`:
- `recall(query: str, k: int = 3, categories: list[str] | None = None) → list[str]`
  — FTS5 RRF via `content_store.search`, plus SessionRAG semantic hits when
  `session_rag.available()`. De-duplicated, budget-trimmed.
- `reflect(...)` — before generation, call `recall()` on `world_state` + joined
  anomalies; pass results in as augmented `recent_events` / a `past_context`
  arg to `inner_monologue.reflect`. **Bounded** by current mode budget so
  injection never exceeds limits (safety rule 2).
- `dream(dry_run=False) → DreamReport` — instantiate DreamCycle, run, return report.
- `session_rag` — lazy property; constructed on first `recall` if Ollama probe
  passes; `None`/skipped otherwise. No exception leaks to reflect.

`context_manager.py`:
- `metabolic_state(used_tokens) → MetabolicState` (delegates to MetabolicContext).
- Optional metabolic line in `ConsciousnessState.to_injection()` (STANDARD/COMPACT only).

`session_lifecycle.py`:
- After `HEARTBEAT_PATH.write_text(...)`, call `engine.dream()` in a
  best-effort `try/except` (handoff already persisted; dream failure must not
  lose the handoff).

## 5. OutputFilter Stages (detail)

- **`DedupBlocks`** — scans lines; runs of an identical line (or identical
  N-line block) collapse to the first occurrence + `… (×N)`. Config: `min_run`
  (default 3), `block_size` (default 1). Use case: stack-trace spam, repeated
  log lines, retry storms.
- **`SecretMask`** — regex redaction of common secret shapes: `sk-[A-Za-z0-9]{20,}`,
  bearer tokens, `AKIA…`, `ghp_…`, `xox[baprs]-…`, generic
  `(api[_-]?key|token|secret|password)\s*[:=]\s*\S+`. Replaces value with
  `***REDACTED***`. Config: extra patterns, on/off. Default **on** for any
  content destined for ContentStore/heartbeat.
- Both honor the crash-safe contract: any exception in a stage returns the
  last-good text. Both get unit tests incl. "does not corrupt non-matching text."

## 7. Safety (unchanged guarantees)

1. No autonomous self-modification — metabolic/Mitosis are advisory only.
2. Injection hard limits — `recall()` results pass through mode budget trimming.
3. Goals advisory — prune happens in Dream (maintenance), not auto-execution.
4. Reflections append-only — Crystallize **archives via new summary + deletes
   originals only after the summary is committed**; never edits a reflection in place.
5. Cannot modify own safety rules — no code path lets Dream/metabolic alter gates.

## 8. Testing & Verification

Target ≈ +60 tests → ≈ 407 total. Per module:

- `test_metabolic.py` (~12) — tier boundaries, usage_pct, advisory flags, edge
  (0 tokens, used > window, window 0).
- `test_dreaming.py` (~12) — each phase in isolation + full run; `dry_run`
  mutates nothing (assert DB row counts unchanged); idempotent second run;
  crystallize threshold respected; relations removed with entity.
- `test_world_model_prune.py` (~6) — `prune_stale` removes age-stale +
  low-relevance + relations; dry_run; persistence across reload.
- `test_event_bus_purge.py` (~5) — `purge_duplicates` hard-deletes only dups;
  `stats` reflects it; idempotent.
- `test_session_rag.py` (~15) — mocked embedder: chunker, cosine ranking,
  re-index delete, `available()` true/false paths, search empty store.
- `test_output_filter_stages.py` (~8) — DedupBlocks runs/blocks/`min_run`;
  SecretMask each pattern + non-match safety + crash-safety.
- `test_perf_reflection.py` (~3) — seed 10k events + 1k entities; assert
  `reflect()` < threshold, `dream()` < threshold; report timings.
- `test_engine_recall.py` (~4) — recall returns FTS5 hits, budget-bounded,
  graceful when RAG unavailable, integrated into reflect.

**Verification discipline (per user mandate):** after each step, beyond running
tests — manually re-read the changed code, check the DB actually changed as
claimed (inspect row counts), confirm no dormant method left unwired, confirm no
safety guarantee weakened. Tests are necessary, not sufficient.

## 10. Out of Scope (YAGNI)

- No FAISS / external vector DB (numpy cosine stays).
- No live token-usage tracking inside Conscio (caller supplies `used_tokens`).
- No "witness-locus", "noetic-helix", "thermodynamic" Noosphere concepts —
  philosophical, no concrete agentic payoff here.
- No auto-firing handoff — advisory only.
- No EntropyTrim / semantic-similarity filter stage this pass (DedupBlocks
  covers the high-value case; revisit if needed).
