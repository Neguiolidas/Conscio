# Conscio v0.8 "Semantic Reconciliation" — Design Spec

> **Date:** 2026-06-06
> **Codename:** Semantic Reconciliation
> **Theme:** Contradiction detection beyond lexical negation + non-destructive semantic output dedup
> **Theory source:** [Claude_Sentience](https://github.com/daveshap/Claude_Sentience) by Dave Shapiro — coherence at the *ontological* scale. v0.6 detected contradiction only by literal negation tokens; v0.8 detects opposition by meaning.
> **Status:** Approved for planning.

---

## 1. Goal

Two upgrades, both gated on the already-present (optional) Ollama embedder used by SessionRAG:

1. **Semantic contradiction** — the ontological coherence dimension currently flags contradiction only when two relation predicates share a lexical core and exactly one carries a negation token (`coherence._relations_contradict`). v0.8 detects semantic opposition: entity states (`operational` vs `offline`) and relations (`owns` vs `sold`) that oppose *in meaning*, with the lexical rule retained as an offline fast-path/fallback.
2. **Semantic output filter** — a new, **non-destructive** `SemanticDedup` filter stage that *annotates* semantically redundant adjacent blocks (never merges/deletes), avoiding the false-merge failure mode that caused v0.6 to reject merge-based semantic dedup.

**One sentence:** *Reconcile the world model semantically during `dream()` (off the hot path), cache the result so the coherence hot path stays cheap, and flag (don't delete) redundant output.*

---

## 2. The honest mechanism (why this is not hand-waving)

Embeddings give **similarity, not polarity** — `cos("operational","offline")` is moderate (same domain), which alone cannot say "these contradict." v0.8 supplies polarity with **embedding-projected antonym axes**:

- An **axis** = a named pair of poles, each pole defined by a few anchor terms: `availability: {+: operational, online, up, running}{−: offline, down, crashed, failed}`.
- Each anchor is embedded once and cached; a **pole vector** = the centroid (mean) of its anchors' embeddings.
- Two assertions A, B **contradict on an axis** iff A projects onto one pole and B onto the opposite pole, each with cosine ≥ `AXIS_THRESHOLD`.
- This **generalizes**: `crashed`, `degraded`, `unreachable` all project to the negative `availability` pole without being in any lexicon — that is exactly what embeddings buy us, used honestly (similarity to a *labeled* pole, not as a fake polarity signal).

Offline / no Ollama → axes unavailable → **fall back to lexical negation only** (current v0.6 behavior, fully preserved).

---

## 3. Architecture

```
dream()  [OFF HOT PATH]   ← all embedding I/O lives here
  └─ Reconcile sub-phase:
        world.mark_contradictions(detector)
          detector = ContradictionDetector(SemanticEngine(embedder, axes))
          → scans relations + entity state_logs
          → writes cached `contradicted` flags into world_model.json

reflect() / coherence.assess()  [HOT PATH — stays cheap]
  └─ ontological_score(world) reads world.contradicted_entities()  (cached, no network)
        via world.list_relations()  (kills the world._data tech debt)

output pipeline  [OPT-IN]
  └─ SemanticDedup stage: annotate near-dup blocks; offline → no-op; never deletes
```

New modules: `conscio/semantic.py`, `conscio/axis_pack.py`, `conscio/presets/axes/*.json`. Touched: `world_model.py`, `coherence.py`, `output_filter.py`, `dreaming.py`, `engine.py`, `docs/noosphere/coherence-engine-model.md`.

`relations_contradict` **reuses** the existing `coherence._strip_neg` / `_relations_contradict` as its lexical layer — single source of truth for negation tokens (no duplication of `_NEG_TOKENS`).

### 4.4 `coherence.ontological_score` refactor (`coherence.py:120-148`)

Replace the private `world._data` read + inline lexical scan with public accessors:
```python
def ontological_score(world) -> float:
    try:
        total = world.entity_count()              # new public accessor
        contradicted = world.contradicted_entities()  # cached, no scan
    except Exception:
        return 1.0
    if total == 0:
        return 1.0
    return _clamp(1.0 - len(contradicted) / total)
```
> **Note:** add `WorldModel.entity_count() -> int` (`return len(self._data["entities"])`) so coherence touches **no** private state — finishing the tech-debt cleanup the v0.6 doc flagged (`world._data` access removed entirely). The lexical inline scan (`_relations_contradict` loop) is **removed from the hot path**; it now lives in `ContradictionDetector` and runs only in dream's Reconcile. On a cold world (never dreamed), `contradicted_entities()` returns `[]` → score 1.0 (no false dissonance before the first reconcile — acceptable and documented).

**Backward-compat path:** v0.7 expected the lexical scan to be available synchronously for its ontological-targeted dream. v0.8 keeps `ContradictionDetector.relations_contradict` callable synchronously (lexical layer needs no network), so v0.7's dream targeting keeps working and simply gets richer when semantics are on.

### 4.5 Dream `Reconcile` sub-phase (`dreaming.py`, `engine.dream()`)

Insert **Reconcile** between Prune and Crystallize:
```
Release → Prune → Reconcile (mark_contradictions) → Crystallize
```
`DreamReport` gains `reconciled_entities: list[str]`. `engine` builds the detector once: `self._contradiction_detector = ContradictionDetector(SemanticEngine(...))` in `__init__`, passed to `DreamCycle.run` (or read off engine). `dry_run` → mark nothing, report the names that *would* be flagged.

This is where v0.7's "ontological-targeted aggressive prune" gets its semantic teeth: the contradicted set it prunes is now the semantic set.

### 4.6 `SemanticDedup` output filter stage (`output_filter.py`)

```python
class SemanticDedup(FilterStage):
    """Annotate semantically redundant adjacent blocks. NON-DESTRUCTIVE:
    never deletes/merges. Offline (no embedder) → returns text unchanged."""
    def __init__(self, semantic=None, threshold: float = 0.88,
                 marker: str = " ↺ near-dup of above ({score:.2f})"):
        ...
    def apply(self, text: str) -> str:
        # split into blocks on blank lines; if embedder unavailable → return text
        # embed blocks; for each block i, if cos(i, i-1) >= threshold,
        #   append marker to block i (keep BOTH blocks verbatim)
    def name(self) -> str: return "semantic_dedup"
```
- Registered in `STAGE_REGISTRY` + handled in `build_stage` (`threshold`, `marker`, optional injected `semantic`).
- **Opt-in, NOT in the default pipeline** — the default reflection pipeline runs on the hot path (`engine.py:106-114`) and must stay network-free. Enabled via engine flag / `CONSCIO_SEMANTIC_DEDUP=1`, wired with the shared `SemanticEngine`.
- Crash-safe by the existing pipeline contract (`FilterPipeline.apply` swallows stage exceptions); the offline no-op makes failures rare anyway.

## 5. Contract notes / deviations

- **Hot path stays cheap.** All embedding I/O is confined to `dream()` (Reconcile) and the opt-in output stage. `coherence.assess` reads only cached flags. This is the central design guarantee and the reason the v0.7+v0.8 pair is safe.
- **Offline-degradable everywhere.** No Ollama → lexical contradiction (v0.6 parity) + `SemanticDedup` no-op. numpy imported lazily so core stays dep-free when semantics are off.
- **Non-destructive output filter** — deliberately *flags*, never merges. Directly answers the v0.6 rejection of merge-based semantic dedup (false-merge destroys content). Recorded in `coherence-engine-model.md`.
- **Cold-world ontology** returns 1.0 until the first dream reconciles — documented; avoids paying for a synchronous scan on every reflect.
- **Custom axes are data, not code** — domain extensibility via `presets/axes/*.json`, mirroring voice presets.

## 6. Testing strategy (RAM-SAFE — per-file only, never full suite)

- `tests/test_semantic_engine.py` — cosine; axis pole centroids; `opposes()` true for cross-pole, false for same-pole/unrelated; `available()` False with no embedder → all-lexical. Uses a **stub embedder** (deterministic vectors), no live Ollama, no network.
- `tests/test_axis_pack.py` — load default `core`; merge multiple packs; missing pack skipped; param>env>default precedence; `available_axis_packs` lists stems.
- `tests/test_contradiction_detector.py` — lexical fast-path (offline); semantic same-core + polarity; axis opposition (`owns`/`sold`); states (`operational`/`offline`).
- `tests/test_world_contradictions.py` — `list_relations` copy semantics; `state_log` cap + on-change-only; `mark_contradictions` writes flags; `contradicted_entities` cheap read; legacy entities (no state_log) safe.
- `tests/test_ontological_semantic.py` — `ontological_score` reads cached flags; cold world → 1.0; no `_data` access; parity with v0.6 lexical when semantics off.
- `tests/test_semantic_dedup.py` — annotates near-dup (stub embedder), keeps both blocks; below threshold → untouched; offline → no-op; registered in `STAGE_REGISTRY`.
- `tests/test_semantic_reconcile_integration.py` — `dream()` Reconcile populates `reconciled_entities`; ontological score improves after a contradiction is resolved (stub embedder).

**Constraint:** never run the full suite or `tests/test_perf_reflection.py` (OOM on the 25GB box). One file per `python3 -m pytest` process. No `-n`, no `--cov`. Stub the embedder in all tests — no test depends on a running Ollama.

## 7. Out of scope / future

- Vector index for axis projection (linear scan over a handful of axes is fine).
- Contradiction *resolution* (v0.8 detects + flags + prunes; it does not auto-rewrite the graph beyond the existing prune).
- Per-relation (not per-entity) contradiction scoring in the coherence aggregate (keep entity-granular for parity with v0.6's denominator).
- Embedding-based output *merge* (explicitly rejected; flag-only).
