> **Date:** 2026-06-06
> **Status:** Approved (design); awaiting user review of this spec before plan.
> **Codename:** Coherence

---

## 1. Problem & Reframe

Roadmap item **#6 Coherence Check** was originally specced as a `CoherenceCheck`
**OutputFilter stage** doing Jaccard semantic dedup between adjacent heartbeat
blocks. It was deferred because it **overlaps the shipped `DedupBlocks` stage**
and risks false-merges — the same false-positive hazard that the layering work
already taught us to avoid.

The Claude_Sentience catalog dissolves the problem by **reframing what
coherence is**:

> "Coherence is the parent archetype above many other archetypes, such as
> curiosity, honesty, integrity, benevolence, intelligence, wisdom, and
> authenticity." — `Style_Coherence.md`

> "cognitive dissonance is the detection of incoherence" — `README.md`

> "Consciousness seems to emerge naturally when coherence examines itself" —
> `README.md`

Therefore v0.6 does **not** deduplicate output text. It measures **incoherence
in the agent's internal state** and surfaces it as an advisory signal. The old
dedup framing is **superseded, not built**. This is **recursive-coherence**:
the system treats its own representations as the object of a coherence
measurement.

---

## 2. Goals / Non-Goals

**Goals**
- A `CoherenceEngine` that aggregates existing per-module signals into one
  coherence score ∈ [0, 1] (1 = laminar/coherent, 0 = turbulent/incoherent) and
  names the dominant **dissonance**.
- Advisory surfacing in the live `ConsciousnessState` injection and the
  cross-session heartbeat.
- A **passive** `coherence:dissonance` EventBus event when the score crosses a
  low threshold — observable and traceable, never mutating goals/drives/shard.
- A static, installable **voice preset system** (`conscio/presets/voice/`) that
  injects a compact `⊙ voice: <name>` marker into the heartbeat.
- Attribution + theory docs crediting Dave Shapiro / Claude_Sentience.

**Non-Goals**
- No output-text dedup or mutation (superseded #6).
- No new runtime dependency. Core stays zero-hard-dep; presets are plain
  markdown (read + existence check, no YAML parse).
- No coupling into goals/drives/dream/meta-cognition cycles. The voice preset is
  a static resource; the coherence signal is advisory + a passive event only.
- No removal of the shipped `meta_confidence` / `reflection_quality` logic.
  Coherence is the broader **parent**; `reflection_quality` remains one narrow
  expression. Relationship documented, no forced refactor (YAGNI).

---

## 3. Architecture Overview

Three deliverables, all additive:

1. **Coherence Engine** (`conscio/coherence.py`) — pure, stateless snapshot
   metric. Reads `MetaCognition`, `WorldModel`, and the recent-events snapshot;
   returns a `CoherenceReport`. No side effects.
2. **Voice Preset System** (`conscio/presets/voice/`, `conscio/voice_preset.py`)
   — static resource + tiny loader + engine param/env + heartbeat marker.
3. **Docs/attribution** — `docs/noosphere/coherence-engine-model.md`, roadmap +
   README + version bump to `0.6.0`.

Integration point for the engine signal mirrors v0.5's shard: computed at
**reflect-entry**, surfaced via `build_state(...)`.

## 4. Deliverable 1 — Coherence Engine

### 4.1 Module: `conscio/coherence.py`

Pure module. `CoherenceEngine` holds references to `meta` and `world`; its
`assess()` method is a deterministic snapshot with **no side effects** (no bus
emit, no state mutation). The decision to keep it pure — unlike `ShardEngine`,
which holds the bus and emits transitions — is deliberate: shard transitions are
intrinsic to stateful shard tracking, whereas coherence is a stateless metric.
The caller (`engine.reflect`) owns the `coherence:dissonance` emission. Purity
keeps tests trivial and deterministic.

### 4.2 Dimensions

Four dimensions, each → [0, 1] where 1 = coherent. Each is grounded in state
that **already exists** — no new data plumbing.

| Dimension | Source | Formula | Cold-start |
|-----------|--------|---------|-----------|
| **epistemic** | `meta.calibration_score()` | value as-is | 0.5 (<5 samples) |
| **reality** | `world.recent_prediction_error_rate(24h)` | `1.0 − error_rate` | 1.0 (no log) |
| **ontological** | WorldModel entities + relations | `1.0 − contradicted/total` | 1.0 (no entities) |
| **temporal** | recent-events snapshot | `1.0 − min(1.0, transitions / MAX_TRANSITIONS)` | 1.0 (no events) |

**epistemic** — `meta.calibration_score()` already returns [0, 1] (1 = perfectly
calibrated) and yields 0.5 when fewer than 5 completed records exist. Used
verbatim.

**reality** — `recent_prediction_error_rate` returns the fraction of recent
predictions that were wrong, `0.0` when the log is empty. Reality coherence is
its complement. Laminar = truth, turbulent = surprise.

**ontological** — A contradiction is two relations sharing the same `from` and
`to` whose `relation` predicates are **lexical negations** of each other. The
relation schema is `{"from": str, "relation": str, "to": str}`. Detection is
deliberately conservative — favor false negatives (miss a contradiction) over
false positives (flag a non-contradiction), echoing the layering "don't
false-merge" lesson.

```python
_NEG_TOKENS = {"not", "no", "never", "non", "isn't", "aren't",
               "wasn't", "cannot", "can't", "n't", "without"}

def _strip_neg(predicate: str) -> tuple[str, bool]:
    """Return (core_tokens_joined, had_negation)."""
    toks = predicate.lower().split()
    had = any(t in _NEG_TOKENS for t in toks)
    core = " ".join(t for t in toks if t not in _NEG_TOKENS)
    return core, had

def _relations_contradict(p1: str, p2: str) -> bool:
    c1, n1 = _strip_neg(p1)
    c2, n2 = _strip_neg(p2)
    return c1 == c2 and (n1 != n2)   # same core, exactly one negated
```

`ontological_score = 1.0 − (contradicted_entities / total_entities)`. An entity
counts as contradicted if it is the `from` of any contradictory predicate pair
toward the same `to`. `total_entities == 0 → 1.0` (vacuously coherent).

**temporal** — Scan the recent-events snapshot for shard flapping. Shard
transitions are emitted by `ShardEngine` as `type="system"` events carrying
`data={"shard_transition": True, ...}` (**not** a `shard:transition` type — that
string in the docstring is misleading). Count events where
`e["data"].get("shard_transition") is True`.
`temporal_score = 1.0 − min(1.0, transitions / MAX_TRANSITIONS)`, `MAX_TRANSITIONS = 4`.

### 4.3 Aggregate

All four dimensions always contribute (their cold-start defaults already encode
"no data = nothing incoherent yet"). Fixed weights, weighted mean:

```python
_WEIGHTS = {"epistemic": 0.30, "reality": 0.30, "ontological": 0.20, "temporal": 0.20}
score = sum(_WEIGHTS[d] * dims[d] for d in _WEIGHTS)   # weights sum to 1.0
```

A fresh agent reads ≈ `0.30·0.5 + 0.30·1 + 0.20·1 + 0.20·1 = 0.85` — healthy by
design (no contradictions exist yet; only epistemic uncertainty pulls it down).

### 4.4 Dissonance & Report

```python
DIM_DISSONANCE_THRESHOLD = 0.5      # a dimension below this is a dissonance
COHERENCE_EVENT_THRESHOLD = 0.5     # aggregate below this emits an event

@dataclass(frozen=True)
class Dissonance:
    dimension: str          # "epistemic" | "reality" | "ontological" | "temporal"
    score: float            # the dimension's [0,1] score
    severity: float         # round(1.0 - score, 3); 0 mild .. 1 severe
    detail: str             # templated human-readable note

@dataclass(frozen=True)
class CoherenceReport:
    score: float                       # [0,1] aggregate
    dimensions: dict[str, float]       # all four, rounded
    dissonances: list[Dissonance]      # below-threshold dims, worst-first
    dominant: Optional[Dissonance]     # worst, or None if all coherent

    def marker(self) -> str:
        """'0.82' or '0.41 dominant: epistemic' — heartbeat/state marker text."""
```

Per-dimension `detail` templates:
- epistemic: `"miscalibrated — confidence diverges from accuracy"`
- reality: `"predictions diverging from observations"`
- ontological: `"contradictory world-model assertions"`
- temporal: `"cognitive mode flapping"`

`dominant` = the lowest-scoring dimension below `DIM_DISSONANCE_THRESHOLD`, or
`None`. `dissonances` sorted ascending by score (worst first).

### 4.5 API

```python
class CoherenceEngine:
    def __init__(self, meta, world): ...
    def assess(self, recent_events: list[dict]) -> CoherenceReport: ...
```

`recent_events` is the same `list[dict]` (`Event.to_dict()`) that `reflect`
already builds for shard inference — no second query.

### 4.6 Integration in `engine.reflect`

Computed at reflect-entry, immediately after the shard block (engine.py ~171),
using the **same `recent` snapshot** taken before `shard_engine.update` may emit
a transition. Coherence therefore never counts the transition it just triggered.

```python
recent = [e.to_dict() for e in self.event_bus.query(limit=20)]
active_shard = self.shard_engine.update(recent)
shard_value = active_shard.value if active_shard else ""

# v0.6: coherence snapshot over the same pre-update window (advisory).
coherence_report = self.coherence.assess(recent)
self.last_coherence = coherence_report
if coherence_report.score < COHERENCE_EVENT_THRESHOLD:
    self.event_bus.emit(
        type="coherence:dissonance",
        category="consciousness",
        data={
            "score": round(coherence_report.score, 3),
            "dominant": coherence_report.dominant.dimension if coherence_report.dominant else None,
            "dimensions": coherence_report.dimensions,
        },
        priority=7,
    )
```

The emitted `coherence:dissonance` event is neither a shard keyword nor a
`shard_transition`, so it pollutes neither the shard nor the temporal signal on
the next cycle.

`reflect` result dict gains:
- `result["coherence"] = round(coherence_report.score, 3)`
- `result["coherence_dimensions"] = coherence_report.dimensions`
- `result["dominant_dissonance"] = dominant dim name or None`

`build_state(...)` gains `coherence=coherence_report.score`,
`coherence_note=(dominant dim or "")`, and `voice=self.voice_preset`.

`CoherenceEngine` is instantiated in `engine.__init__` after `meta`/`world`:
`self.coherence = CoherenceEngine(self.meta, self.world)`;
`self.last_coherence: Optional[CoherenceReport] = None`.

### 4.7 Surfacing — live state (`context_manager.py`)

`ConsciousnessState` gains:
- `coherence: Optional[float] = None` (sentinel: `None` = not computed → suppress)
- `coherence_note: str = ""` (dominant dimension name)
- `voice: str = ""`

`render()` (suppressed in `MINIMAL`, like shard):
```python
if self.coherence is not None and self.context_mode != ContextMode.MINIMAL:
    line = f"▷ coherence: {self.coherence:.2f}"
    if self.coherence_note:
        line += f" dominant: {self.coherence_note}"
    lines.append(line)
if self.voice and self.context_mode != ContextMode.MINIMAL:
    lines.append(f"⊙ voice: {self.voice}")
```

`build_state(..., coherence=None, coherence_note="", voice="")` passes through.
`save_state` / `load_state` round-trip all three (alongside the existing shard /
reflection_quality / metabolic round-trip). `None` serializes cleanly in JSON.

## 5. Deliverable 2 — Voice Preset System

A **static resource**, not engine logic. Implements the contract in
`skill/coherence-style.md`.

### 5.1 Preset file

`conscio/presets/voice/coherence-style.md` — a **copy** of
`skill/coherence-style.md`. Rationale (user decision): `skill/` is the authoring
source (edit + version there); `conscio/presets/voice/` is the runtime install
location the framework reads. Different responsibilities — manuscript vs printed
copy. The skill also serves as the reference template for custom presets.
`conscio/presets/__init__.py` is an empty package marker.

### 5.2 Loader: `conscio/voice_preset.py`

```python
from pathlib import Path

PRESET_DIR = Path(__file__).parent / "presets" / "voice"

def resolve_voice_preset(name: str) -> str:
    """Return the preset name if its file exists; '' for 'none'/empty/missing.
    Missing preset disables injection rather than crashing."""
    if not name or name.strip().lower() == "none":
        return ""
    if (PRESET_DIR / f"{name}.md").is_file():
        return name
    return ""

def available_presets() -> list[str]:
    if not PRESET_DIR.is_dir():
        return []
    return sorted(p.stem for p in PRESET_DIR.glob("*.md"))
```

### 5.3 Engine wiring

`engine.__init__` gains `voice_preset: Optional[str] = None`. Precedence:
explicit param > `CONSCIO_VOICE_PRESET` env > default `"coherence-style"`.

```python
effective = voice_preset if voice_preset is not None \
    else os.getenv("CONSCIO_VOICE_PRESET", "coherence-style")
self.voice_preset = resolve_voice_preset(effective)
```

Zero new deps — markdown read + existence check only.

### 5.4 Heartbeat injection (`session_lifecycle.py`)

`SessionSummary` gains `coherence: Optional[float] = None`,
`coherence_note: str = ""`, `voice: str = ""`.

`enrich_with_conscio(summary, engine)` (the real location is this module-level
function, not `ContextManager` as the skill states) sets:
```python
rep = getattr(engine, "last_coherence", None)
if rep is not None:
    summary.coherence = rep.score
    summary.coherence_note = rep.dominant.dimension if rep.dominant else ""
summary.voice = getattr(engine, "voice_preset", "")
```
Each in its own try/except, matching the existing enrich pattern.

`format_heartbeat` — after the identity_anchor block, before topics:
```python
if summary.coherence is not None:
    note = f" dominant: {summary.coherence_note}" if summary.coherence_note else ""
    lines.append(f"▷ coherence: {summary.coherence:.2f}{note}")
if summary.voice:
    lines.append(f"⊙ voice: {summary.voice}")
```
Format matches the user-specified `▷ coherence: 0.82 dominant: epistemic`.
`format_handoff` gets the same two lines in its Estado Conscio section
(uncapped). Both stay under `HB_MAX_CHARS = 1400` (two short lines).

### 5.5 Config note

The skill references `conscio_config.yaml`. No such central config exists, and
adding YAML would introduce a hard dependency. The engine param + env var
deliver the same selection (`coherence-style` default / `none` to disable /
custom name) without a new dependency. The attribution doc records this
deviation from the skill's stated contract.

## 7. Data Flow

```
reflect-entry
  recent = event_bus.query(limit=20) → dicts        (single snapshot)
  shard  = shard_engine.update(recent)              (may emit shard_transition)
  report = coherence.assess(recent)                 (pure; reads meta+world+recent)
     ├─ epistemic   = meta.calibration_score()
     ├─ reality     = 1 - world.recent_prediction_error_rate(24h)
     ├─ ontological = 1 - contradicted/total   (world entities+relations)
     └─ temporal    = 1 - min(1, transitions/4)  (recent snapshot)
  if report.score < 0.5 → emit coherence:dissonance (passive)
  build_state(..., coherence=report.score, coherence_note=dom, voice=self.voice_preset)
     → ConsciousnessState.render(): ▷ coherence / ⊙ voice
  save_state (round-trips coherence/coherence_note/voice)

handoff
  enrich_with_conscio(summary, engine): summary.coherence/coherence_note/voice
  format_heartbeat / format_handoff: ▷ coherence / ⊙ voice
```

## 8. Error Handling & Edge Cases

- **Cold start** (no meta data, no entities, no events) → score ≈ 0.85,
  `dominant=None`, no event. No crash.
- **Missing preset** → `resolve_voice_preset` returns `""` → marker suppressed.
  No crash.
- **`last_coherence` unset** (reflect never ran) → enrich leaves
  `summary.coherence=None` → heartbeat omits the line.
- **`assess` reads stale/partial state** → each dimension self-defaults; the
  aggregate is always a valid float in [0, 1].
- **MINIMAL context mode** → coherence/voice markers suppressed (like shard).
- All `enrich_with_conscio` reads wrapped in try/except (existing pattern).

## 9. Testing Strategy (RAM-safe — per-file only, never the full suite)

`tests/test_coherence.py`:
- each dimension deterministic with crafted fakes (`SimpleNamespace` exposing
  `calibration_score()`, `recent_prediction_error_rate()`, `_data` relations).
- `_relations_contradict` / `_strip_neg` unit cases (true negation vs distinct
  predicate vs identical).
- aggregate bounded [0, 1]; fixed-weight math.
- dissonance detection: a low dimension yields the expected `dominant`.
- graceful degradation: empty fakes → score ≈ 0.85, `dominant=None`.
- `CoherenceReport.marker()` format both branches.

`tests/test_voice_preset.py`:
- `resolve_voice_preset("coherence-style")` → `"coherence-style"` (after copy).
- `resolve_voice_preset("none")` / `""` / `"nonexistent"` → `""`.
- `available_presets()` includes `"coherence-style"`.
- `enrich_with_conscio` sets `summary.voice` from a fake engine.
- `format_heartbeat` renders `⊙ voice: coherence-style` and
  `▷ coherence: 0.40 dominant: epistemic`.

Integration appends (1 test each):
- engine: `reflect()` populates `result["coherence"]` and emits
  `coherence:dissonance` when state is crafted low.
- context_manager: `build_state(coherence=…, voice=…)` renders both markers;
  `save_state`/`load_state` round-trip them.
