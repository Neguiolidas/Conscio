# Conscio v0.4 "Phenomenological Consciousness" — Self-Judgment

> **Status:** Design approved 2026-06-05. Scope: focused functional trio (roadmap modules #2, #3, #7).
> Derived from [Noosphere-Manifold](https://github.com/acidgreenservers/Noosphere-Manifold) (CC BY-NC-SA 4.0) — operational paraphrase, not verbatim copy.

## Goal

Give the agent the ability to **judge its own world-model and its own reflections** before acting on them. Each of the three modules adds a deterministic judgment gate to an existing loop:

- **Entropy** judges *entities* — connectivity can rescue an old node; isolated faded nodes die.
- **Friction** judges *reflections* — a reflection is not crystallized into "truth" if the world moved on from it.
- **Meta-reflect** judges *the reflection itself* — emits a quality signal that closes the Witness loop (Generate → Observe → **Analyze** → Learn → Apply).

Non-goals for v0.4 (deferred): Shard Engine (#1), Content Layering (#4), Trajectory Vector (#5), Coherence Check (#6). These either require schema migration, a signal source the EventBus does not yet provide, or duplicate an existing stage.

## Constraints

- **No schema migration.** ContentStore/EventBus tables unchanged.
- **No new dependencies.** Pure Python + stdlib + existing SQLite.
- **WorldModel stays pure-JSON.** No SQLite coupling introduced into WorldModel.
- **Advisory, never executing.** Internal signals (entropy, friction, meta_confidence) advise; they never fire an irreversible action on their own.
- **Backward compatible.** Existing methods and their tests stay green. New behavior is opt-in.
- **RAM-aware.** Test count kept modest; no large fixtures beyond the existing perf guard.

## Module A — Entropy-aware World Model

File: `conscio/world_model.py`. All functions deterministic and pure (no I/O beyond the existing `_save`).

### A1. `entropy(name) -> float`

Returns an entropy score in `[0, 1]`. Higher = more disordered = better prune candidate.

```
age_days     = (now - last_updated).total_seconds() / 86400          # >= 0
age_norm     = 1 - exp(-age_days / HALFLIFE_DAYS)                     # [0,1), HALFLIFE_DAYS = 7
relation_cnt = number of relations touching `name`
isolation    = 1 - min(relation_cnt / MAX_RELATIONS, 1.0)            # [0,1], MAX_RELATIONS = 8
rel_gap      = 1 - clamp01(relevance)                                # [0,1]
entropy      = clamp01( W_AGE*age_norm + W_ISO*isolation + W_REL*rel_gap )
```

Weights (must sum to 1.0): `W_AGE = 0.4`, `W_ISO = 0.3`, `W_REL = 0.3`. Defined as module constants so they are tunable in one place.

Edge cases:
- Unknown entity name → returns `1.0` (maximally disordered; nothing to keep).
- Unparseable `last_updated` → `age_norm = 1.0` (treat as maximally old).
- Missing `relevance` → defaults to `1.0` (so `rel_gap = 0.0`), consistent with the rest of WorldModel.

Connectivity rescue: a 30-day-old entity with 8+ relations and relevance 0.9 →
`age_norm ≈ 0.986, isolation = 0.0, rel_gap = 0.1` → `entropy ≈ 0.4*0.986 + 0 + 0.3*0.1 = 0.42` → **survives** a 0.85 threshold. A 1-day-old isolated entity with relevance 0.05 →
`age_norm ≈ 0.133, isolation = 1.0, rel_gap = 0.95` → `entropy ≈ 0.4*0.133 + 0.3 + 0.3*0.95 = 0.64` → survives 0.85 but is clearly more disordered; pushing relevance to ~0 and age a few days crosses 0.85.

### A2. `prune_by_entropy(threshold=0.85, dry_run=False) -> list[str]`

1. Real run only: call `decay_all_entities()` first (so `relevance` is current), exactly as `prune_stale` does.
2. Compute `entropy(name)` for every entity. dry_run projects the decay in-line (recompute decayed relevance without persisting) so the preview matches a real run — the same fidelity discipline applied to `prune_stale` in v0.3.
3. Collect names with `entropy > threshold`.
4. Real run: `remove_entity(name)` for each (drops relations + saves). dry_run: return the list, mutate nothing.

Returns the list of names removed (or that would be removed). `prune_stale` and its 7 tests are left untouched and remain available for any other caller.

### A3. `recently_changed(hours=24) -> set[str]` *(prereq for Friction)*

Returns the set of entity names whose `last_updated` is within the last `hours`. ~5 lines; mirrors the parsing in `stale_entities`. Unparseable timestamps are excluded (a node with a broken timestamp is not "recently changed").

### A4. Prediction-error tracking

Feeds Module C. Stored as a **sliding-window list** inside the world JSON under a new top-level key `"prediction_log"` (added to `_load`'s default skeleton for forward-compat).

`record_prediction(entity, expected_state, actual_state) -> bool`
- `error = 1 if expected_state != actual_state else 0` (exact string compare; `surprise = bool(error)`).
- Append `{"entity": entity, "expected": expected_state, "actual": actual_state, "error": error, "ts": now_iso()}`.
- **Bound the log on every append** (prevents JSON inflation, caveat #1):
  1. Drop entries with `ts` older than `PREDICTION_LOG_RETENTION_HOURS = 168` (7 days).
  2. If still longer than `PREDICTION_LOG_MAX = 500`, keep the newest 500.
- `_save()`. Returns `surprise`.

`recent_prediction_error_rate(window_hours=24) -> float`
- Consider entries with `ts` within `window_hours`.
- Return `sum(error) / count` over that window, or `0.0` if the window is empty.

> SQLite for prediction history is the future path if per-session volume ever outgrows a 500-entry / 7-day window; not needed at current scale and would couple WorldModel to a DB it does not own today.

## Module B — Friction in DreamCycle

File: `conscio/dreaming.py`. New phase ordering: **Release → Prune → Friction → Crystallize**.

### B1. Prune step switches to entropy

`DreamCycle.run` Prune step calls `engine.world.prune_by_entropy(threshold=self.prune_entropy_threshold, dry_run=dry_run)` instead of `prune_stale`. New constructor param `prune_entropy_threshold: float = 0.85`. The old `prune_min_relevance` / `prune_max_age_hours` params are kept on the constructor (accepted, no longer used by the Prune step) so existing construction sites and tests do not break; documented as deprecated.

The list of pruned names is captured and handed to Friction.

### B2. `_friction(engine, pruned) -> set[int]`

Returns the set of reflection `source_id`s to **defer** (skip this cycle).

```
changed = set(pruned) | engine.world.recently_changed(24)
candidates = old reflections selected by _crystallize's cutoff query
deferred = set()
for row in candidates:
    text = concatenated chunk text for row (already fetched in _crystallize)
    if any(name in text for name in changed):
        deferred.add(row["id"])
return deferred
```

Notes:
- Deterministic substring membership (`name in text`) — no NLP, no thresholds.
- `changed` is small (pruned + last-24h), so the per-candidate scan is cheap.
- dry_run: friction is computed but nothing is deleted (consistent with the rest of dry_run).

### B3. Crystallize honors the defer set

`_crystallize` is refactored to:
1. Select candidates (existing cutoff query).
2. Compute `deferred = self._friction(engine, pruned)`.
3. Build the summary from **non-deferred** candidates only.
4. Delete **only non-deferred** originals (deferred reflections survive intact for a future cycle, where the world may have settled).
5. The append-only safety from v0.3 is preserved: index the summary first, capture `summary_id`, never delete `summary_id`.

If fewer than `crystallize_min_count` **non-deferred** candidates remain, crystallization is skipped this cycle (deferred ones are not lost — they remain for next time).

### B4. Report

`DreamReport` gains `reflections_deferred: int = 0` (and in `to_dict`). `reflections_consolidated` continues to count what was actually crystallized.

## Module C — Meta-reflect

File: `conscio/engine.py` (`reflect()`), `conscio/context_manager.py` (state surface). Advisory only.

### C1. Compute `meta_confidence`

After `monologue.reflect(...)` returns, in `engine.reflect`:

```
error_rate      = self.world.recent_prediction_error_rate(window_hours=24)   # [0,1]
anomaly_pen     = min(0.1 * len(anomalies), 0.5)                              # capped at 0.5
meta_confidence = clamp01( confidence * (1 - error_rate) * (1 - anomaly_pen) )
```

Properties: bounded `[0,1]`; never exceeds the input `confidence`; monotonically decreasing in both `error_rate` and anomaly count. Worked example: `confidence=0.8, error_rate=0.25, anomalies=2` → `0.8 * 0.75 * 0.8 = 0.48` → MEDIUM.

Label: `HIGH` if `>= 0.66`, `MEDIUM` if `>= 0.33`, else `LOW`.

### C2. Persistence and visibility

- **Persist** on the existing reflection EventBus event: add `meta_confidence` (rounded) to its `data` dict — already emitted, queryable, no FTS pollution. *(No separate ContentStore row: the reflection text is already indexed; meta_confidence is a quality signal, not searchable content.)*
- **Return**: add `meta_confidence` and `reflection_quality` (the label) to `reflect()`'s result dict.
- **Visibility**: `ConsciousnessState` gains a `reflection_quality: str = ""` field, rendered as one line in `to_injection()` (suppressed in MINIMAL mode, matching the `metabolic` line pattern). `build_state` accepts a `reflection_quality` kwarg; `reflect` passes the label.

`meta_confidence` does **not** feed back into drives or goals — advisory only, per the safety rule.

## Testing

Pure-logic and small-fixture tests only (RAM-aware). New files / additions:

- `tests/test_world_model_entropy.py`
  - `entropy` monotonic in age, isolation, relevance; bounded `[0,1]`; unknown-name → 1.0; bad-timestamp → high.
  - `prune_by_entropy`: rescues old-but-connected; kills isolated-faded; dry_run matches real run after decay; returns removed names.
  - `recently_changed`: includes fresh, excludes old/unparseable.
- `tests/test_world_model_prediction.py`
  - `record_prediction` returns surprise flag; mismatch vs match.
  - sliding window: retention drops >7d entries; hard cap at 500; rate over 24h window; empty window → 0.0.
- `tests/test_dreaming_friction.py`
  - friction defers a reflection mentioning a pruned/recently-changed entity.
  - friction allows a reflection with no overlap.
  - empty changed-set → nothing deferred; deferred reflections are NOT deleted.
- `tests/test_engine_meta_reflect.py`
  - `meta_confidence` bounded, never exceeds `confidence`, decreases with anomalies and error_rate.
  - correct label thresholds; `meta_confidence` present in result dict and on the reflection event data.
  - `reflection_quality` surfaced in injection (non-MINIMAL).

Existing suites must stay green, especially `tests/test_world_model_prune.py` (7) and `tests/test_dreaming.py`.
